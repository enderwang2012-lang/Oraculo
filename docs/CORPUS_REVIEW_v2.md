# 语料审查 v2：为何旧库偏「鸡汤」

## 现象

App 内置 `phrases.json` 来自 `feikuai_corpus_v1.csv`（30）+ `feikuai_corpus_v1_lab.csv`（20）。用户体感「太鸡汤」，与 Oraculo（纯色 + 留白 + 触动感）气质不符。

## 根因（三层）

### 1. 入库策略放大了「可取单 ≠ 代表灵魂」

`feikuai_generation_spec.md` 为**星巴克取单口令**设计了两层门：

- **第一层**：能不能念出口、会不会社死 → 大量「好运签 / 轻励志 / 模板疗愈」可通过。
- **第二层**：再分 `anchor` / `active` / `lab`。

`lab` 本意是「可取单但不进灵魂上限」，但 `scripts/embed_corpus.py` **把 v1 与 lab 全部打进 App**，等于 40% 语料是宪法里明确标为 borderline 的句子（宇宙、治愈、你值得、光明在前方、勇敢向前冲等）。

### 2. 母题权重偏向「安抚 + 保证」

旧库 `emotion_theme` 分布偏重：

| 母题 | 问题句型 |
| --- | --- |
| `light_comfort` | 一切都会好的、治愈正在发生、你值得被温柔 |
| `soft_hope` | 美好/光明在前方、你终将遇见光、好运马上就来 |
| `self_affirmation` | 你已经够好、你值得世间美好 → 模板式肯定 |
| `daily_romance` | 人间皆是浪漫、生活值得热爱、世界温柔待你 |

这些在宪法 **§否决规则** 里属于：说太满、只有结论、缺少投射空间。

### 3. 与 Oraculo 产品场景不一致

啡快规范允许「冲鸭」「人间值得」；Oraculo 是**无人念出口的静屏 oracle**，更接近 `feikuai_corpus_constitution.md` 的**灵魂上限**：

- 轻、准、近、留白
- 过程而非结论
- 画面与动作，而非「美好」「治愈」「值得」

## v2 筛选原则（Oraculo 专用）

每条须同时满足：

1. **四问**（宪法）：像人话、有情绪落点、留投射空间、值得停一眼。
2. **硬否决**：宇宙/命运/治愈一切/你值得/终将/美好在前方/加油冲鸭/人生圆满/硬命令。
3. **触感**：优先具体物象（雨、灯、门、茶、袖口、台阶）或未完成过程（未拆、将起、还凉）。
4. **长度**：主句 4–8 字为主，极少数 9 字须有节奏必要。
5. **分层**：仅 `anchor` + `active`，**不收录 lab**。

## 保留与淘汰（v1 摘要）

| 保留进 v2 | 淘汰原因 |
| --- | --- |
| 不必赶、先缓一缓、此刻够用了、轻轻放过 | 轻安顿，留白 |
| 风会转向、再等一小会儿、路边有光、风里有春天 | 过程/画面，非保证 |
| 不必再证明、先走再说、慢慢走也行 | 体面、低推动 |
| — | 一切都会好的、你已经够好、惊喜正在路上 → 好运签/模板肯定 |
| — | 全部 `feikuai_corpus_v1_lab.csv` → 仅 lab 级 |
