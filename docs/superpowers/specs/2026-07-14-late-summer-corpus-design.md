# 夏末入秋语料扩展设计

## 范围

本轮为 Oraculo 生成一批新的中文候选语料，以夏末入秋为主、常青内容为辅。工作先止于研究、候选生成和用户评审，不直接修改主语料、翻译、派发标签、语料版本或线上静态资源。

用户确认的三条种子语料直接进入候选池：

- `窗外的麻雀`
- `孤独海怪`
- `囿于昼夜`

这三条标记为“用户提供·截断致敬”，不标记为原创。

## 目标

- 交付 90 条可评审候选，其中 63 条属于夏末入秋窗口，27 条为常青内容。
- 提高语料的歌词性、诗性和语言密度，减少泛化的鼓励、治愈和社交媒体文案感。
- 保持 3-8 字的自然错落，以 4-6 字为重心；只有无法继续删减的句子可以达到 9 字，且不超过 3 条。
- 允许约 20%-30% 的候选带孤独、困顿、荒诞或轻微暗色，不强制转成正向结论。
- 让原创候选脱离参考作品后仍能独立成立，并为所有截断致敬项保留来源记录。

## 非目标

- 不围绕某一位词作者或音乐人做高相似度仿写。
- 不保存、汇编或批量摘录完整歌词。
- 不在候选阶段改动 `starbucks_now_passphrases.csv`、翻译表、情境标签或 freshness 元数据。
- 不在本轮设计内自动晋升候选、提升语料版本、提交远端或发布 CDN。
- 不把“诗性”“留白”等写作原则变成面向用户的语料分类。

## 语言方向

本轮采用三种语言机制混合，而不是复用“民谣”题材符号。

### 生活切片

以可见、可听、可触的普通事物为抓手，通过位置、动作或声音形成情绪。候选应接近一个被截下来的生活镜头，而不是完整叙事。

### 陌生化组合

连接语义距离较远的词，例如情绪与动物、时间与器物、身体与天气。组合必须能产生可感知的张力；只为晦涩而生造的词组直接淘汰。

### 抽象压缩

用动词、介词、时间概念或空间关系压缩一段更长的经验。句子可以保留谜面，但朗读和基本语义必须成立。

三种机制在原创候选中按约 4:3:2 分布。它们是内部研究字段，不用于 Markdown 评审稿的分组标题。

## 内容构成

### 时间窗口

夏末入秋内容通过闷热将退、蝉声渐疏、骤雨、晚风、夜色、果实、返程和衣物变化等迹象暗示时令。默认不直接出现“八月”“立秋”“处暑”等时间标签，除非删掉标签会破坏句子的核心张力。

常青内容重点补充独处、城市夜晚、普通物件、轻微荒诞、不完整情绪和低强度的人际连接。它们不承担节气派发任务，可以长期保留。

### 原创与致敬

90 条候选中，截断致敬最多 9 条：

- 三条用户种子固定保留。
- 最多再加入六条有明确来源、截断后仍适合 Oraculo 的短片段。
- 如果找不到足够好的致敬片段，空余名额由原创补足，不为比例凑数。

所有致敬项都记录作品、创作者、来源链接和风险说明。风险标记只用于内容治理，不代表完成法律授权或版权判断。未经用户在评审阶段逐条选中，致敬项不得晋升主语料。

## 参考研究

研究表至少覆盖 20 首华人创作歌曲、8 位创作者，并辅以至少 6 首现代诗和 4 组古典诗词意象。参考范围可以横跨民谣、摇滚和创作型流行歌词；选择依据是语言机制，而不是音乐流派标签。

每项参考只记录以下内容：

- 普通名词如何获得情绪或象征作用。
- 地点和物件如何形成场所感。
- 两个语义层级不同的词如何建立关系。
- 现实叙述转入抽象的具体节点。
- 原表达删减后，哪些结构仍保留张力。

研究文件不得保存整段歌词。只有进入致敬候选的短片段可以原样记录，并必须同时记录来源。

研究输出保存为 `review/corpus_references_2026_08.md`。

## 生成与筛选

### 原始草稿

先生成至少 150 条原创草稿，再与致敬种子一起筛选为 90 条正式候选。生成分三路独立进行，避免所有句子共享同一种节奏和意象。

### 第一轮：去文案感

默认淘汰以下内容：

- 口号、完整祝福、心理安慰话术和正向结论。
- `把 X 交给 Y`、`X 也温柔`、`好事正在路上` 等高频模型句法。
- 仅依赖“风、云、月、山、远方”等诗意词汇堆叠的空画面。
- 过度对称、为了整齐而写成的四字结构。
- 无法解释内部关系、只剩陌生感的生造组合。

`微光`、`奔赴`、`答案`、`治愈`、`慢慢`、`终会`、`热爱` 等高频词不是绝对禁词，但出现时必须有不可替代的具体关系，且不得在本批中反复使用。

### 第二轮：相似性复核

- 对 `starbucks_now_passphrases.csv` 做标准化后的精确去重。
- 对候选池内部做标准化去重和高重合短语检查。
- 人工复核重复意象、同构句法和仅替换一个名词的变体。
- 对照研究记录回看原创候选；接近原作标志性表达的句子必须淘汰或改标为致敬候选。

本轮不新增 embedding 或运行时依赖。相似性复核是离线内容治理步骤。

### 第三轮：朗读检查

每条候选都要回答四个问题：

1. 朗读是否自然，断句和轻重是否顺畅？
2. 是否至少有一个具体抓手，或者一个真正成立的陌生关系？
3. 脱离上下文后能否独立成立？
4. 它是否让人停顿，而不是像海报文案一样被立即读完？

任何一项不成立的候选都不进入正式评审稿。质量不足时重新生成草稿，不用低质量句子补齐 90 条。

## 评审产物

### Markdown 评审稿

`review/corpus_candidates_2026_08_late_summer.md` 包含：

- 1-90 的打散编号。
- 候选文字。
- 致敬项的必要来源说明。

原创机制、时间窗口和暗色标记不作为分组标题展示，避免影响用户的直觉判断。

### CSV 评审表

`review/corpus_candidates_2026_08_late_summer.csv` 包含以下字段：

- `candidate_no`
- `phrase`
- `content_window`
- `mechanism`
- `tone_shade`
- `source_type`
- `source_work`
- `source_creator`
- `source_url`
- `risk_note`
- `status`
- `review_note`

`tone_shade` 使用 `light` 或 `dark`，只用于核对暗色候选比例，不在 Markdown 评审稿中展示。`source_type` 使用 `original`、`user_seed` 或 `homage_excerpt`。初始 `status` 统一为 `pending`。

用户选中后，晋升前的唯一输入文件为 `review/corpus_candidates_2026_08_accepted.csv`。未选中的候选不会进入主库。

## 晋升与发布边界

用户完成候选选择后，后续实施才包括：

1. 对入选项做最终原创性和来源复核。
2. 补充英文翻译。
3. 生成主题、情境、颜色和 freshness 标签。
4. 使用现有候选晋升脚本写入主语料。
5. 重建本地语料资源并运行验证。

至少运行以下验证：

```bash
python3 scripts/validate_corpus.py
python3 scripts/validate_dispatch.py
python3 scripts/validate_phrase_freshness.py
python3 -m unittest tests/test_tag_phrases_rules.py tests/test_phrase_freshness_tags.py
```

候选生成阶段不运行发布命令。提交远端和线上热更新必须再次获得用户明确授权。

## 验收标准

- 评审稿恰好包含 90 条候选，编号连续且 CSV 与 Markdown 一致。
- 夏末入秋与常青内容分别为 63 条和 27 条。
- 三条用户种子原样出现，并标记为 `user_seed`。
- 截断致敬不超过 9 条，且没有缺失来源的致敬项。
- 至少 70% 的候选为 4-6 字；其余候选形成 3、7、8 字的自然错落，9 字候选不超过 3 条。
- 暗色候选占比在 20%-30%，但不集中使用同一类孤独或困顿意象。
- 与现有主语料和候选池内部没有标准化精确重复。
- 没有大面积重复的词根、句法骨架或情绪结论。
- 所有原创候选都能脱离参考作品独立成立。
- 候选阶段只新增研究和评审文件，不改动主语料或发布资源。
