# 啡快式情绪语库生成与筛选规范

## 目标

这份规范用于指导 AI 稳定生成候选句，并通过统一标准筛掉“像，但没魂”的句子。

原则：

- 先大量生成
- 再严格筛选
- 再小幅润色
- 最后才进入正式语库

不要让模型直接输出“最终答案”，而要把它当作高通量候选生产器。

## 生产流程

建议采用四段式流程：

1. `母题设定`
2. `批量生成`
3. `逐句筛选`
4. `入库复核`

## 第一步：母题设定

每轮生成前，必须先固定以下参数：

- `emotion_theme`
- `intent`
- `tone`
- `scene`
- `energy_level`
- `era`
- `batch_size`

示例：

- `emotion_theme`: `light_comfort`
- `intent`: `soothe`
- `tone`: `calm`
- `scene`: `decision_wait`
- `energy_level`: `low`
- `era`: `daily_softness`
- `batch_size`: `80`

如果不先定母题，模型会混出一堆方向不稳的句子。

## 第二步：批量生成

### 基本要求

- 一次生成 `60-120` 条候选
- 不要求首轮就漂亮
- 先追覆盖面，再追精度

### 生成硬约束

- 默认 `4-8` 个汉字
- 句子必须可单独展示
- 不要写解释
- 不要用引号
- 不要编号
- 禁止重复已有核心短语

### 推荐生成提示词骨架

```text
你正在为一套“轻触灵魂”的移动端情绪短语库生成候选句。

目标不是写漂亮文案，而是写出人在普通一天里愿意轻轻相信的话。

本轮参数：
- 情绪母题：{{emotion_theme}}
- 功能目标：{{intent}}
- 语气：{{tone}}
- 场景：{{scene}}
- 能量等级：{{energy_level}}
- 风格年代：{{era}}

要求：
1. 每句 4-8 个汉字，极少数可略放宽
2. 不说教，不命令，不喊口号
3. 不要“宇宙奖励你”“终将闪耀”这类廉价鸡汤
4. 优先使用生活动作、轻画面、柔和状态
5. 句子要像人话，像一句被递过来的小回应
6. 允许留白，不要把意义说死
7. 不要直接复写以下已有句子：{{blocked_examples}}

输出 {{batch_size}} 条，只输出句子本身，每行一条。
```

### blocked examples

每轮都应放入一批“禁止直接复写”的高频锚点。

例如：

- `心缓自有答案`
- `答案在路上`
- `重要的是出发`
- `给生活一点留白`
- `慢下来看世界`
- `做你自己就很好`

这样模型会沿着灵魂走，而不是抄表皮。

## 第三步：逐句筛选

筛选比生成更重要。

**关键前提：先换位到取单场景。**

想象店员在门口念口令、顾客走过来接杯、后面还有人排队——这句话能不能被自然念出口？顾客是会觉得「有点意思 / 被轻轻点到 / 今天抽到的签还行」，还是「尬 / 像广告 / 不想接」？

真实啡快样本里同时存在：`奥利给`、`人间值得`、`以秋风之名`、`记得微笑`、`美好正在路上`、`冲鸭冲鸭`。  
所以「口语化」「好运签感」「轻劝诫」「结构换皮」本身**不是**淘汰理由；它们是这个品类的正常语法。

筛选应分两层，不要混用。

### 第一层：取单场景门（Pass / Hard Reject）

先问：**这句话作为取单口令，能不能用？**

#### 硬否决（仅此几类）

出现以下任一情况，才直接 `reject`：

- **超长**：常规口令超过 7 个汉字（联名/IP 特例另论）
- **拗口**：念出来断句奇怪、像病句、像把两个 slogan 粘在一起
- **取单社死（坏的那种）**：让人真难堪、被贴标签、被评判，而不是「可爱地害羞」
- **强广告感**：像促销、像品牌宣言、像功能说明
- **真实样本撞车**：与 `starbucks_now_passphrases.csv` 重复，或与高频锚点句仅一字之差复刻
- **硬命令/硬审判**：`必须…`、`别再…` 且语气像上司训话（注意：软劝诫不算，见下）

#### 不再作为硬否决项（常见误判）

以下写法在真实啡快样本中大量存在，**不能**因为「像鸡汤 / 像励志 / 像伪诗」就一票否决：

| 误判标签 | 真实样本参照 | 正确处理方式 |
| --- | --- | --- |
| 廉价疗愈 | `人间值得`、`给你温暖阳光` | 降层到 `lab`，或整批限量 |
| 轻励志 | `冲鸭冲鸭`、`奥利给`、`悄悄惊艳大家` | 可取单，`active` / `lab` |
| 好运签换皮 | `美好正在路上`、`好事将至`、`好消息加载中` | 品类语法，看整批密度 |
| 轻劝诫 | `记得微笑`、`事缓则圆`、`不必逞强` | 可取单，偏 `active` |
| 季节文艺 | `以秋风之名`、`一杯敬月光` | 可取单，偏 `lab` / `active` |
| 网络口语 | `Hi走啦`、`来了都是打工人` | 可取单，偏 `lab` |

#### 取单场景朗读测试（必做）

每条候选至少念一遍，模拟店员报号：

- 「您好，**[口令]**，您的拿铁好了。」

如果念完你自己愿意接这杯，第一层通常可过。

### 第二层：语库分层（anchor / active / lab）

通过第一层后，再按**取单场景四维**打分，单项 1-5 分：

- `speak_score`（可念性）
  念出口顺不顺；有没有自然停连；是否像会被当面念出来的话。
- `sign_feel`（签感）
  像不像「今天随机抽到的一句小签 / 小身份 / 小祝愿」。
- `delivery_score`（情绪送达）
  接杯瞬间有没有被轻点到、被逗到、被安慰到——不要求文学性。
- `distinct_score`（辨识度）
  是不是烂大街模板；同批内是否结构扎堆。

总分建议：

- `17-20`：入 `anchor`（少而精，决定气质）
- `14-16`：入 `active`（日常主力，最像真实啡快的中位风格）
- `11-13`：入 `lab`（可取单，但模板化、略尬或略熟）
- `10` 以下：即使第一层勉强能过，也不进正式库

### 与「宪法标准」的关系

`feikuai_corpus_constitution.md` 描述的是**语库灵魂上限**：更克制、更留白、更轻触。

生成与筛选时：

- **取单场景门**决定「能不能用」
- **宪法标准**决定「值不值得进 anchor / 能不能代表产品灵魂」

同一句可以是：

- 取单场景：`pass`
- 语库分层：`lab`（例如 `惊喜正在路上`）
- 宪法评价：结构正确，但新鲜度一般

这不算矛盾。

### 筛选提示词骨架

```text
请你审核以下候选句，假设它们会出现在移动端取单口令场景：店员会当面念出来，顾客走过来接杯。

第一层：取单场景门
- 只 hard reject：超长、拗口、坏社死、强广告、与已有真实样本重复、硬命令训话
- 不要因为“像鸡汤/像励志/像伪诗/像好运签换皮”就直接淘汰——真实啡快样本里大量存在这类句子

第二层：取单场景四维打分（1-5）
- 可念性 speak_score
- 签感 sign_feel
- 情绪送达 delivery_score
- 辨识度 distinct_score

分层：
- anchor / active / lab / reject

reject 只留给第一层 hard reject 或第二层总分 ≤10 的句子。
```

## 第四步：入库复核

入库前再做三件事：

- `去重`
  检查是否与已有句仅一字之差、同义改写、结构完全重复。
- `成组检查`
  看整批是否同质化过高。
- `朗读测试`
  读出来是否自然，是否有拗口或“写出来像、念出来假”的问题。

## 推荐产出格式

生成后的中间表建议至少包含这些列：

- `id`
- `text`
- `emotion_theme`
- `intent`
- `tone`
- `scene`
- `energy_level`
- `speak_score`
- `sign_feel`
- `delivery_score`
- `distinct_score`
- `scenario_gate`
- `final_bucket`
- `editor_note`

## 质量红线（取单场景版）

以下不是「见到就杀」，而是「知道风险、控制密度」。

### 1. 模板好运签

例：

- `惊喜正在路上`
- `好运马上就来`
- `美好就在前方`

问题：
不是不能用——真实样本里 `美好正在路上` 就是这一类。  
风险是**整批扎堆**，会让语库像同一个模板批量打印。

处理：
单句可进 `active` / `lab`；同结构在同批里不超过 3 条。

### 2. 熟语疗愈

例：

- `宇宙会拥抱你`
- `你值得被温柔`
- `一切都会好的`

问题：
单独念出口并不违和，但太熟、太空，难当 anchor。  
更像真实啡快的中位签，而不是灵魂句。

处理：
优先 `lab`；只有可念性特别好时才升 `active`。

### 3. 压迫式励志

例：

- `再难也要奋力向前`
- `今天必须行动`
- `相信自己必胜`

问题：
这类在取单场景里会像被督促，顾客接杯压力上升。  
与 `冲鸭冲鸭` 的差别在于：后者有玩梗缓冲，前者像打卡。

处理：
倾向 hard reject；除非改成口语玩梗（如 `冲鸭冲鸭` 路线）。

### 4. 书面伪诗意

例：

- `月光轻拂破碎灵魂`
- `在晨曦里回收心事`

问题：
不是「文艺就一定不行」——`以秋风之名` 可以，`破碎灵魂` 不行。  
分界点：**念出来像不像人在说话**。

处理：
- 像季节/歌词/画面：`lab` / `active`
- 像作文标题：hard reject

### 5. 硬命令劝诫

例：

- `别再犹豫了`
- `你必须开心`

问题：
软劝诫在真实样本里成立：`记得微笑`、`事缓则圆`。  
硬命令像训话，取单场景会尬。

处理：
- 软：`lab` / `active`
- 硬：hard reject

## 推荐的生成节奏

如果要建设第一版正式语库，建议节奏如下：

1. 每个母题先生成 `200-300` 条候选
2. 每个母题保留 `30-60` 条
3. 五大母题先做出 `180-250` 条正式库
4. 联名/节日/特殊场景另建侧库

这比一口气生成一千条再硬挑更稳。

## 版本策略

建议未来维护三个版本：

- `v1_core`
  最稳、最像灵魂中心的一批
- `v1_extended`
  可扩展日常库
- `v1_experimental`
  有趣但尚未完全稳定的实验句

这样产品上线时可以只用 `v1_core + v1_extended`，实验句单独观察。

## 总纲

生成这套语库时，最重要的不是“写出很多句子”，也不是“把每句都筛成文学作品”，而是：

`先保证它像一句会被念出来的取单口令，再决定它值不值得进 anchor。`

句子不需要炫技。
它需要在店员念出来、顾客接走那杯咖啡的半秒钟里，刚刚够用。
